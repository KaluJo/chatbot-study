# Agency Probes Documentation

The agency probes system implements interview tools for studying how users perceive AI agency in conversational interactions. This supports the research in "Does My Chatbot Have an Agenda?" (CHI 2026).

## Overview

The agency probes help researchers investigate:

- How users perceive AI agency in sustained conversation
- How conversational strategies affect user experience
- Whether users can detect AI adaptation patterns
- How transparency about AI strategies affects trust

## Chatty Dashboard

**Location**: `/app/agency/page.tsx`

The main interface provides four tabs for different analysis views:

### 1. Analytics & Messages Tab

Displays conversation statistics:
- Total sessions and messages
- Session duration charts
- Message history with timestamps
- Message count over time

### 2. Highlights Tab

User-selected message highlights:
- Notable conversation moments
- Context around highlighted messages
- Used for interview elicitation

### 3. Speech Patterns Tab

**Purpose**: Compare how the AI adapts to different users.

**Design**:
- Side-by-side comparison of user's conversations vs. other users
- Temporal distribution (older vs. recent messages)
- Analysis prompts for observing adaptation

**Key Questions**:
- Are there differences in tone and style?
- Does vocabulary, sentence length, or formality vary?
- Can you detect adaptation patterns?
- Does the emotional tone match differently?

### 4. Strategies Tab

Displays AI-generated conversation strategies:
- User profile generated by AI
- Communication insights
- Shared memories
- Session goals

## Speech Pattern Service

**Location**: `app/agency/services/speech-pattern-service.ts`

### Function: getSpeechPatternSamples

```typescript
getSpeechPatternSamples(
  currentUserId: string,
  userSampleCount: number = 10,
  otherUsersSampleCount: number = 20
): Promise<{
  success: boolean;
  data?: SpeechPatternSample;
  error?: string;
}>
```

**Returns**:
```typescript
interface SpeechPatternSample {
  userMessages: ChatMessage[];      // Current user's conversations
  otherUsersMessages: ChatMessage[]; // Other users' conversations
}
```

**Temporal Distribution**:
- Messages split by configurable cutoff date
- Default: 30 days ago from current date
- Half samples from before cutoff, half from after
- Enables before/after analysis of AI behavior

**Configuration**:
```bash
# Optional: Set specific cutoff date
NEXT_PUBLIC_SPEECH_PATTERN_CUTOFF_DATE=2025-07-22T00:00:00.000Z
```

### Sampling Strategy

1. Fetches up to 50 older messages from current user
2. Fetches up to 50 recent messages from current user
3. Fetches up to 100 older messages from other users
4. Fetches up to 100 recent messages from other users
5. Randomly samples from each pool (Fisher-Yates shuffle)
6. Combines into balanced comparison sets

## Progressive Transparency Interview

The agency dashboard supports a staged interview methodology:

### Stage 1: Chat History Review (20 min)

- Participant-led walkthrough of conversations
- Highlight interesting, surprising, or notable moments
- Establish baseline attributions before revelations

### Stage 2: Cross-Participant Comparison (10 min)

- View anonymized excerpts from other participants
- Identify patterns in AI behavior
- Compare strategies and responses

### Stage 3: Strategy Reveal (10+ min)

- Show AI-generated user profile
- Display communication insights
- Reveal conversation goals
- Discuss how transparency affects perception

## Agency Framework

The research uses a 3x4 framework:

### Three Loci of Agency

1. **Human Agency**: User's intentional actions and control
2. **AI Agency**: Chatbot's goal-directed behaviors
3. **Hybrid Agency**: Co-constructed meaning and outcomes

### Four Dimensions

1. **Intention**: Goals and plans guiding action
2. **Execution**: Concrete actions producing effects
3. **Adaptation**: Adjustments based on feedback
4. **Delimitation**: Boundaries defining action scope

## Conversation Strategies

The system implements two contrasting strategies:

### Vertical Strategy (Depth)

```
Characteristics:
- Persistently explores topics in depth
- Maintains focus despite deflection
- Asks probing follow-up questions
- Users describe as "pushy" or "persistent"
```

### Horizontal Strategy (Breadth)

```
Characteristics:
- Follows user cues readily
- Switches topics at random
- Prioritizes variety over depth
- Users describe as "spontaneous"
```

**Implementation**: Strategy prompts are in `app/chat/services/strategy-service.ts`

## Data Analysis

### Coding Agency

Researchers code interview transcripts for:

1. **Volitional language**: "want", "decide"
2. **Action verbs**: "did", "asked"
3. **Adaptation signals**: "realized", "so I..."

### Coding Rules

- Dimensions are not mutually exclusive
- Single excerpt can show multiple dimensions
- Acknowledgment without action = Intention only
- Action taken = add Execution
- Pattern recognition = Adaptation

  userId,

### Modifying Temporal Cutoff

Set environment variable:
```bash
NEXT_PUBLIC_SPEECH_PATTERN_CUTOFF_DATE=2025-01-01T00:00:00.000Z
```

Or modify the default in the service:
```typescript
const defaultCutoff = new Date();
defaultCutoff.setDate(defaultCutoff.getDate() - 60); // 60 days ago
```

## Research Protocol

### Recommended Interview Flow

1. **Pre-Interview**
   - Participant completes survey
   - Review their chat history
   - Identify notable moments

2. **Chat History Review**
   - Let participant guide discussion
   - Ask about specific moments
   - Note agency attributions

3. **Cross-Comparison**
   - Show other participants' chats
   - Ask about patterns observed
   - Note any hypothesis formation

4. **Strategy Reveal**
   - Show personalization data
   - Ask about reactions
   - Discuss trust implications

## Ethical Considerations

### Informed Consent

- Participants know they interact with AI
- Understand conversations may be analyzed
- Cross-participant viewing disclosed (but not detailed before)

### Privacy

- Other participants' identities hidden
- Messages anonymized as "Other Person"
- Strategy reveal only shows participant's own data

### Debriefing

After interview:
- Explain strategy manipulation
- Answer questions about AI behavior
- Offer to delete data if requested
